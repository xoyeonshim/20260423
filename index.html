02 / PROBLEM
문제 정의

HR 커뮤니케이션,
매일 같은 일을 반복하고 있었다.

퇴직자 안내, 복직 예정자 조직 확인, 기간제 계약만료 안내, 자산 반납 리마인드…
정형화된 메시지를 매일 다른 대상에게 보내는 일이 하루 업무의 상당 부분을 차지했습니다.

PAIN-01

반복되는 수동 발송

매일 대상자를 확인하고, 이름·날짜·링크를 바꿔가며 슬랙·메일을 개별 발송. 10분이 쌓여 하루가 되는 일.

DAILY
PAIN-02

놓치기 쉬운 타이밍

복직 예정자의 조직 확인, 계약 만료 D-day 안내 등 정확한 시점에 발송되어야 하는 업무. 깜빡하면 곧장 운영 이슈로.

RISK
PAIN-03

복잡한 조건 분기

1년 미만 / 1년 이상 퇴직자, 당일 퇴직 / 사전 신청, 일반 / 기간제… 케이스마다 문구와 안내 링크가 달라짐.

COMPLEX
PAIN-04

누락 없는 기록 필요

누구에게, 언제, 어떤 안내를 보냈는지 추적 가능해야 함. 수동 발송은 로그가 사람 기억에 남음.

04 / FEATURES
구현 기능

5개 플로우,
2개의 트리거.

자동 트리거(주중 오전 10-11시)와 수동 메뉴 트리거(범위 지정 팝업)를 조합해,
정형 업무는 알아서 돌고, 유연성이 필요한 업무는 클릭 한 번으로 실행됩니다.

01
퇴직자 셀프체크리스트 리마인더
근무 종료일 당일 자산 반납·서류 제출 안내를 자동 발송. 체크박스 기준으로 대상 추출.
AUTO
02
복직 조직 문의 안내
복직 예정일이 다가온 구성원의 조직 확인을 담당 리더에게 요청. 수식 기반 자동 대상 선별.
AUTO
03
기간제 계약 만료 안내
1년 미만 / 1년 이상 조건 분기 자동 적용. 슬랙 + HTML 메일 동시 발송, BCC로 Payroll 팀 자동 참조.
MANUAL
04
기간제 자발적 퇴직 안내
당일 퇴직 / 사전 신청 분기. 팝업에서 대상 행·퇴직 유형 선택 → 본문 자동 생성.
MANUAL
05

05 / IMPACT
Before & After

같은 업무,
다른 리듬.

같은 정보를 전달하는 업무이지만, 걸리는 시간과 남는 흔적이 완전히 달라졌습니다.

Before

하루를 시작하는 일이
"오늘 누구한테 뭐 보내야 하지?"

  • 매일 아침 시트 열어 대상자 확인
  • 이름·날짜 바꿔가며 슬랙 DM 복붙
  • 조건별로 다른 메일 템플릿 찾기
  • 발송 기록은 '아마 보냈던 것 같은' 수준
  • 깜빡하면 구성원이 먼저 질문으로 찾아옴
일 평균 소요 시간
~ 45분
After

시트를 채우면,
알림은 알아서.

  • 평일 오전 자동 트리거로 대상자 자동 판별
  • 조건 분기(재직기간·퇴직유형) 자동 본문 생성
  • 슬랙 DM + Gmail HTML 메일 동시 발송
  • 담당자 채널에 성공·실패 리포트 자동 공유
  • Q열 자동 완료 체크로 이중 발송 방지
일 평균 소요 시간
~ 3분

    muteHttpExceptions: true
  };
  const res = UrlFetchApp.fetch(url, options);

}

08 / LEARNINGS
과정에서 배운 것

부딪혔고,
배웠다.

AI 코드라고 다 되는 건 아니었습니다. 실제 환경의 제약을 하나씩 풀어가는 과정이 핵심이었습니다.

01

Enterprise Slack의 토큰 스코프 이슈

일반 Slack 봇 토큰으로는 DM 발송이 제한되는 경우가 있었습니다. Workspace Admin과 협업해 필요한 권한(chat:write, users:read.email)만 최소 범위로 부여.

SOLUTION Script Property에 토큰을 저장하고, 조직 내 공식 채널로 배포 → 보안과 재사용성 확보
02

성능 — getRange 반복 호출의 비용

행마다 getValue를 여러 번 호출하니 속도가 눈에 띄게 느려짐. AI에게 구조 리팩토링을 요청해 "행 단위 batch read"로 전환.

SOLUTION sheet.getRange(row, 1, 1, maxCol).getValues()[0] 로 1회 읽기 → 발송당 소요시간 60%↓
03

이중 발송 방지 로직

자동 트리거와 수동 발송이 공존하다 보니 동일 대상자에게 메시지가 두 번 나가는 리스크가 있었음.

SOLUTION 발송 성공한 행만 Q열에 true 자동 기록 → 다음 트리거에서 자동 제외
04

조건 분기의 복잡도 관리

1년 미만 / 1년 이상, 당일 / 사전 퇴직 등 케이스가 늘어날수록 본문이 얽힘. 템플릿을 함수로 분리하고 입력값으로만 제어.

SOLUTION sendEmail_fixedterm(name, email, date, isUnderOneYear) — 재사용 가능한 함수 시그니처
